1. Konsep Logika & Proposisi

Dalam informatika, semua instruksi adalah logika. Proposisi adalah unit dasar logika berupa kalimat pernyataan yang bernilai Benar (True) atau Salah (False). Nilai ini tidak boleh keduanya atau ambigu.

Proposisi Tunggal vs Majemuk

Proposisi Tunggal: Kalimat deklaratif tunggal tanpa kata penghubung logis. Contoh: "Biner adalah angka."

Proposisi Majemuk: Gabungan proposisi tunggal menggunakan operator logis (AND, OR, NOT). Contoh: "CPU memproses data DAN RAM menyimpannya."

Hukum De Morgan & Penalaran

Hukum De Morgan: Sangat vital dalam pemrograman. Negasi dari (A Dan B) adalah (Bukan A) Atau (Bukan B). Sebaliknya, negasi (A Atau B) adalah (Bukan A) Dan (Bukan B).

Penalaran Deduktif: Proses menarik kesimpulan dari premis umum ke khusus yang pasti benar secara logis. Contoh: Jika semua Linux adalah Open Source, dan Ubuntu adalah Linux, maka Ubuntu adalah Open Source.

2. Manajemen & Analisis Data

Proses mengubah data mentah menjadi informasi yang bermanfaat bagi user atau pengambilan keputusan.

Data Kualitatif vs Kuantitatif

Data Kualitatif: Data subjektif, deskriptif, berupa kata-kata atau kualitas.
Contoh: "Performa Linux lebih stabil dibandingkan OS X."

Data Kuantitatif: Data objektif berupa angka (numerik) yang dapat dihitung.
- Diskret: Angka bulat (hasil menghitung), misal: Jumlah CPU core.
- Kontinu: Angka desimal/rentang (hasil mengukur), misal: Kecepatan processor 3.5 GHz.

Sumber Data: Primer vs Sekunder

Data Primer: Data tangan pertama. Diambil langsung oleh peneliti melalui pengumpulan orisinal (Sensus penduduk, Wawancara, Observasi langsung).

Data Sekunder: Data yang sudah disediakan pihak lain. Peneliti hanya menganalisis (Laporan tahunan perusahaan, Data historis saham dari API Bursa, Artikel Jurnal, Website pemerintah).

Informasi & Makna

Data mentah seperti "175cm" tidak memiliki arti tanpa konteks. Setelah dianalisis dan diberi label (Tinggi Badan Siswa), ia menjadi Informasi. Gabungan informasi yang dipahami mendalam menjadi Pengetahuan.

3. Struktur Data Terstandar

Cara efisien untuk mengatur data di memori komputer.

Stack (Tumpukan - LIFO)

LIFO (Last In First Out): Elemen terakhir yang masuk akan keluar pertama. Seperti tumpukan piring atau baju di koper.

  • Operasi: Push (Tambah), Pop (Ambil).
  • Penerapan: Fitur Undo/Redo, Navigasi Back/Forward browser, Proses Pemanggilan Fungsi (Recursion).

Queue (Antrean - FIFO)

FIFO (First In First Out): Elemen pertama yang masuk akan keluar pertama. Seperti antrean beli tiket.

  • Operasi: Enqueue (Masuk Antrean), Dequeue (Keluar Antrean).
  • Penerapan: Antrean Printer, Penjadwalan Tugas CPU, Antrean Pelanggan Restoran.

4. Perangkat Keras & Model IPO

Komputer modern mengikuti Arsitektur Von Neumann.

Model IPO (Input-Process-Output)

Input: Data masuk melalui Keyboard, Mouse, Scanner.
Process: Dilakukan oleh CPU (Central Processing Unit). CPU memiliki ALU (Aritmatika/Logika) dan CU (Kontrol).
Output: Hasil ditampilkan di Monitor, Printer, Speaker.
Storage: Data disimpan di HDD/SSD (Permanen) atau RAM (Sementar/Volatile).

CPU Bits: Operasi Aritmatika & Logika. ALU adalah bagian CPU yang menghitung operasi matematika dasar dan perbandingan logis.

5. Sistem Bilangan & Konversi

Komputer hanya mengenal sinyal elektrik (On/Off = 1/0).

  • Biner (Radiks 2): Angka 0, 1. Dasar semua data digital.
  • Desimal (Radiks 10): Angka 0-9. Basis angka manusia.
  • Oktal (Radiks 8): Angka 0-7.
  • Heksadesimal (Radiks 16): Angka 0-9, A-F (A=10, B=11, s/d F=15).

Konversi: Perlu dikuasai untuk memahami bagaimana warna (RGB) dan alamat memori (IP Address) ditulis.

6. Jaringan & Konektivitas

Sistem komunikasi antar komputer.

  • LAN: Local Area Network (Satu gedung).
  • WAN: Wide Area Network (Antar kota/negara/Internet).
  • Topologi: Bentuk fisik jaringan (Star, Mesh, Ring).
  • Protokol: Aturan bahasa dalam jaringan (IP Address, HTTP, TCP).
